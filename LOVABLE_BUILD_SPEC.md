### 3.2 Three-Panel Functionality

**Left Panel: Available Keywords**
- Search bar with real-time filtering
- Filters: Intent, Min Volume, Max Difficulty, Opportunity Score
- Infinite scroll with cursor pagination
- Each keyword shows: text, volume, opportunity score
- Checkbox for multi-select
- "Assign to Thread" button when selected
- **Suggested Clusters** section at bottom (AI-grouped keywords)

**Center Panel: Threads**
- Draggable/reorderable thread cards
- Each thread shows:
  - Name (editable inline)
  - Priority badge (P1-P5)
  - Status badge (Draft/Confirmed)
  - Keyword count and metrics
  - Expandable keyword list
  - Topics list
- Keywords within thread are draggable between threads
- "Add Thread" button

**Right Panel: Thread Details**
- Appears when thread selected
- Editable fields:
  - Thread name
  - Status dropdown
  - Priority dropdown
  - Custom Instructions (structured form):
    - Strategic Context (textarea)
    - Differentiation Points (tag input)
    - Content Angle (text)
    - Target Audience (text)
    - Additional Notes (textarea)

### 3.3 Drag & Drop Interactions

| Interaction | Behavior |
|-------------|----------|
| Drag keyword from left panel to thread | Assigns keyword to thread |
| Drag keyword between threads | Moves keyword to new thread |
| Drag keyword within thread | Reorders keyword (visual only) |
| Drag thread up/down | Reorders threads |
| Click cluster "Assign to Thread" | Opens modal to select/create thread |

### Virtual Scrolling

**Critical for keyword tables with 10,000+ rows:**
```typescript
// Use TanStack Virtual or react-window
import { useVirtualizer } from '@tanstack/react-virtual';

const virtualizer = useVirtualizer({
  count: keywords.length,
  getScrollElement: () => parentRef.current,
  estimateSize: () => 48, // row height
  overscan: 10

```

### Optimistic Locking

All PATCH requests include `version` field. Handle 409 conflicts:

```typescript
const updateThread = async (id: string, data: Partial<Thread>) => {
  try {
    await api.patch(`/threads/${id}`, { ...data, version: thread.version });
    // Optimistically update UI
  } catch (error) {
    if (error.response?.status === 409) {
      // Show conflict dialog
      showDialog({
        title: "Update Conflict",
        message: "This item was modified. Reload and try again?",
        actions: [
          { label: "Reload", onClick: refetch },
          { label: "Cancel", onClick: dismiss }
        ]
      });
    }
  }
};
```

### Cursor Pagination

Use cursor-based pagination for keyword lists:

```typescript
const { data, fetchNextPage, hasNextPage } = useInfiniteQuery({
  queryKey: ['keywords', strategyId],
  queryFn: ({ pageParam }) =>
    fetchKeywords(strategyId, pageParam),
  getNextPageParam: (lastPage) => lastPage.pagination.next_cursor,
});
```

### Error Handling

- Toast notifications for API errors
- Retry failed requests (3 attempts with backoff)
- Graceful degradation when data unavailable
- Loading skeletons during fetch
